Emergency Procedures

üö® Emergency Stop ‚Äì Immediate Action

Trigger: something looks very wrong ‚Äì runaway losses, repeated errors, strange trades.

  1. Stop Engine Immediately
    • In Engine Terminal: press CTRL + C.
  2. Check Pending Orders 
    python scripts/check_pending_orders.py
    • Confirm no unknown/unwanted orders ‚úì
  3. Flatten All Positions (if needed) 
    python scripts/emergency_flatten.py --all --market
    • Confirm all positions closed on broker platform ‚úì
  4. Check Damage 
    python scripts/show_pnl.py --today
  5. Review What Happened 
    tail -n 200 logs/alpha_prime.log
    • Understand cause before restarting ‚úì

üî¥ Kill Switch Activated (Automatic)

If built‚Äëin risk kill switch triggers, logs may show:

CRITICAL: KILL SWITCH ACTIVATED
CRITICAL: Daily loss limit reached: -‚Çπx,xxx
INFO: All strategies paused
INFO: Closing open positions...
INFO: Engine entering SAFE mode

When you see this:

  • Do NOT restart engine immediately ‚úì
  • Confirm all positions closed (broker platform + show_pnl.py) ‚úì
  • Review logs to understand cause ‚úì
  • Decide if trading should stop for the day ‚úì
  • If restarting another day, adjust risk or disable problem strategy ‚úì

üü° Broker Connection Lost

From Command Terminal:

python scripts/reconnect_broker.py

If still down:

  • Log into broker web platform, manually manage open positions ‚úì
  • Do not restart engine until connection stable ‚úì
